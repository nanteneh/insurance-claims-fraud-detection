# Insurance Claims Fraud Detection Application using Tkinter and Machine Learning

A desktop application that detects fraudulent insurance claims using machine learning.
Upload a CSV dataset, train three models simultaneously, and export flagged claims — all through a simple GUI.

## Hardware & Software Requirements

**Minimum Hardware:**
- Processor: Intel Core i5 or equivalent
- RAM: 8 GB (16 GB recommended for large datasets)
- Storage: 500 MB free disk space
- Display: 1280×720 resolution

**Software:**
- OS: Windows 10/11, macOS 10.15+, or Linux (Ubuntu 20.04+)
- Python: Version 3.9 or newer
- Libraries: `pandas`, `numpy`, `scikit-learn`, `xgboost`, `matplotlib`, `scipy`, `tkinter`

## Installation Guide

**Step 1 — Install Python**

Download Python 3.9+ from [python.org](https://python.org). Make sure `pip` is included during installation.

**Step 2 — Install Required Libraries**
```bash
pip install pandas numpy scikit-learn xgboost matplotlib scipy
```

**Step 3 — Download the Application**

Clone or download this repository to get `insurance_claims_fraud.py`.
```bash
git clone https://github.com/nanteneh/insurance-claims-fraud-detection.git
```

## Main Features

- **CSV File Upload** — Browse and load your insurance claims dataset directly in the GUI.
- **Fraud Analysis** — Trains three models on your data (Logistic Regression, Random Forest, XGBoost) and displays fraud/non-fraud counts, Accuracy, AUC-ROC, and F1-score for each.
- **Fraud Results Export** — Saves a `fraud_results.csv` file in the current directory listing claim IDs flagged by each model and those flagged by all three.
- **Visualizations** — ROC curves for side-by-side model comparison.
- **Fraudulent Claims List** — Top 10 flagged claims shown in the GUI; full results saved in `fraud_results.csv`.

## Cautions & Warnings

### Input CSV Requirements

Your CSV must include the following columns:
```
months_as_customer, age, policy_number, policy_bind_date, policy_state, ..., fraud_reported
```

> A template CSV has been provided in this repository.

**Critical Notes:**
- ❌ Do **NOT** include a `claim_id` column — it is auto-generated by the program.
- ✅ The `fraud_reported` column must exist with values `Y` (fraud) or `N` (non-fraud).
- 📅 Dates must be in `MM/DD/YYYY` or `YYYY-MM-DD` format.
- Replace missing values with `?` or leave the cell blank.

**Performance Notes:**
- Datasets over 100,000 rows may cause slow performance.
- Close other memory-intensive applications before running.

## Troubleshooting

| Error | Cause | Fix |
|-------|-------|-----|
| `KeyError: 'fraud_reported'` | Missing column in CSV | Add `fraud_reported` column with `Y`/`N` values |
| `ValueError: time data "..." doesn't match format` | Wrong date format | Use `MM/DD/YYYY` or `YYYY-MM-DD` |
| `MemoryError` | Insufficient RAM | Reduce dataset size or upgrade hardware |
| Missing module errors | Libraries not installed | Run `pip install --force-reinstall pandas numpy scikit-learn xgboost matplotlib scipy` |
